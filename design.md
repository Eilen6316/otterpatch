# 智能体工作台 · 设计文档(design.md)

> 通用 Office 办公套件的 Agent 自动化平台 · 全开源不付费
> 状态:🔒 已锁定 / 🟡 待定 / ⭐ 关键

---

## 1. 产品定义

**通用 Office 办公套件(及更广)的 Agent 自动化平台——让任何"办公自动化的想法"都能成真。**

形态:左侧渲染 Office 文档(Excel/Word/…),用户用**鼠标圈选区域 + 自然语言**表达意图,Agent 据此修改;改动以**可审阅 diff**(逐块接受/拒绝/回滚)落地。让任何人(含新手)用自然语言"完美驱动"Office。

参考形态:Next AI Draw.io(左编辑器 + 右 Agent 面板),但加上 drawio 没有的**空间圈选**与**外科手术式 diff**。

**两层表述(分受众)**:
- **愿景(对所有人)**:让任何人用自然语言"完美驱动"Office,把办公自动化的想法变成现实。
- **技术身份(对开发者/开源圈)**:面向真实 Office 文件的**可审阅安全执行层**——Agent 不直接改文件,只提交结构化修改建议,经校验/影子/diff/人审/rebase/高保真写回后才落盘(**像给 Office 文件提 PR**)。

---

## 2. 战略与原则 🔒

- **全开源 · 不付费**(纯 FOSS)。成功指标 = 被人用、社区繁荣。
- **推理成本走 BYOK + 本地模型** → 项目自身零推理成本。
- **通用是产品定义,聚焦是上市顺序**:平台愿景通用,但第一刀先用一个能惊艳、能可靠的杀手用例(圈选 Excel/Word→说想法→可审阅 diff)打穿,再靠适配器+技能横向扩张。
- **首发格式:Excel + Word**(Excel 先跑通闭环,Word 同期作第二适配器校验抽象层)。
- **国际化(i18n)= 中文 + 英文双语**:UI、Agent 系统提示词、技能描述(用于检索)全部双语;从第一天就把文案抽进 i18n 资源,不写死。
- **部署形态:本地优先桌面应用(Electron,msi/exe/dmg)为主 + 可选自托管 Web(同一套 Node 后端);不做托管 SaaS**。理由:高保真引擎(LibreOffice 子进程)需本地运行、文件隐私不出域、契合 BYOK/本地模型/信创气隙。LibreOffice 与本地模型作为首启可选下载组件,核心安装包不臃肿。
- **UI 克制不花哨、功能齐全**:左编辑器 + 右 Agent 驾驶舱,简洁度参考 Next AI Draw.io;深/浅色;中英双语。

---

## 3. 核心洞察

1. **交互原语**:圈一块区域 + 说想法 → Agent 改。解决 chat-only 办公 AI 的 **grounding 问题**(Agent 不知道你指哪)。
2. **圈选 = 替用户补上缺失的专业词汇**:新手说不清"B2:D7 的条件格式",但能圈住那块说"把这堆数字弄好看点"。
3. **"完美驱动" = 可靠性是生死线**:被改坏的文件 = 信任崩塌,新手不会修。
4. **核心价值 = 格式无关抽象层 + 技能生态 + 治理/审计 + 可私有自托管**。
5. **文档 = 从语义模型编译/投影的产物**:Agent 改语义,不改像素。
6. **公开保真 benchmark 本身就是护城河**:系统性公开"引擎 × 特性 × 写回策略 × 存活率"——没人敢做,做了即成行业参照、立信誉、可传播(见 `kill-experiments.md`)。

## 4. 架构:格式无关抽象层 + 异构底座适配器 🔒

> 没有单一开源底座能同时满足"高保真打开真实文件 + 圈选可寻址锚点 + Agent 可编程改写 + 自由 markup" → **按格式选底座,用自研抽象层统一**。这层抽象是核心 IP。

```
┌──────────────────────────────────────────────┐
│ 接地交互:圈选/红笔标注(指哪)+ 自然语言(要啥)  │
├──────────────────────────────────────────────┤
│ Agent 内核:技能检索 → 计划 → 受约束 ChangeSet    │
├──────────────────────────────────────────────┤
│ ★ 格式无关抽象层(核心 IP):                      │
│   语义锚点 · ChangeSet · 可审阅 diff · 技能 · 回滚 │
├──────────────────────────────────────────────┤
│ 底座适配器:Excel→Univer | Word→TipTap | PPT→Univer│
├──────────────────────────────────────────────┤
│ 高保真引擎(自托管免费·子进程):LibreOffice + 外科补丁│
├──────────────────────────────────────────────┤
│ 模型接入:BYOK / 本地模型(Ollama 等)/ MCP 双向   │
└──────────────────────────────────────────────┘
```

markup 圈选层一律**自建覆盖层**(绝对定位 SVG/Canvas)解耦,不依赖底座内置。

> 📐 **抽象层完整 TS 接口规格见 [`abstraction-layer.md`](abstraction-layer.md)**(锚点 / ChangeSet / Diff / 技能 / 适配器契约 / 并发事务 / 写回保真)。

## 5. 底座选型 🔒

> 受许可证(MIT/Apache,见 §9)约束:只用 permissive 底座,**放弃 AGPL 的编辑器**(OnlyOffice/PPTist/Collabora 嵌入);**高保真不丢**——改由 **LibreOffice 子进程 + 外科补丁** 拿到。

| 格式 | 底座 | 协议 | 要点 |
|---|---|---|---|
| **Excel** | **Univer** 做编辑交互 + **LibreOffice headless(子进程)** 做免费高保真往返 | Apache + MPL/LGPL | Range 一等可寻址对象,圈选→改写最顺;自带 MCP+Skills;LibreOffice 子进程调用许可安全 |
| **Word** | **TipTap/ProseMirror**(推荐)或 **Univer Docs**(备) | MIT / Apache | TipTap:文本 range 锚点 + Decoration 做圈选/批注/diff overlay 最佳、最成熟;Univer Docs:与 Excel 同框架但未成熟、.docx 导出在 Pro。两者背后均用 LibreOffice 做 .docx 高保真 |
| **PPT**(后续) | **Univer Slides** | Apache | 与 Univer 同框架;不用 AGPL 的 PPTist |
| **高保真引擎** | **LibreOffice headless(子进程)** | MPL/LGPL | 真实 .docx/.xlsx 高保真渲染/往返,自托管免费;子进程调用不触发 copyleft |
| **写回保真** | **外科手术 OOXML 补丁** | 许可中立 | 只改目标部件 XML、其余字节原样;不碰引擎代码,MIT/Apache 下更是核心手段 |

## 6. 核心交互闭环 🔒

**圈选(接地)→ 自然语言(意图)→ Agent 先报计划(透明)→ 可审阅 diff(逐块接受/拒绝/撤销)→ 高保真引擎校验后提交。**

- 锚点:像素只是瞬时,持久化的是语义锚点(Excel=单元格地址、Word=文本 range/相对位置),随文档 mutation 自动平移抗漂移。
- Agent 输出 = **挂起的建议事务**,人审通过才并入主文档(可审阅,且防 Agent 高频改动淹没协同)。

### 界面:渐进披露驾驶舱(克制默认,按需展开)

UI 克制简洁(参考 Next AI Draw.io),**简洁是默认态、强大是按需展开**:默认只露"选区上下文 + 说想法";Agent 报计划后才展开计划/待提交摘要/PR 式 diff 卡片/接受拒绝;提交记录默认折叠在底部。

```
┌ 顶栏: 文件名 · ●已保存 · 模型▾ · ↶撤销↷ · 历史 ───────────────────────────┐
├────────────────────────────────────────────┬──────────────────────────────┤
│ 工具条: ⭕圈 ➡️箭头 ✏️删 〰️重写 ❓问  缩放    │  🤖 Agent 驾驶舱               │
│ ┌────────────────────────────────────────┐ │  ① 你圈的:销售数据 12行5列      │ ← 圈选即显示(人话)
│ │     ╭ 圈选 B2:F20 ╮  真实 Excel/Word 高保真│ │  ② 说想法 / 红笔标注…    [发送] │ ← 默认只露这两段
│ │   ┌ E5 1200→📊+环比 ┐ ✓✗  (inline diff)  │ │ ┄┄ 报计划后才展开 ┄┄┄┄┄┄┄┄ │
│ │   └──────────────────┘                   │ │  ③ 我打算做 4 件事 ▾           │
│ └────────────────────────────────────────┘ │  ④ 待提交:改18格·+1列公式·5样式│
│                                            │ │  ⑤ diff 卡片(像 PR,含原因)   │
│                                            │ │  ⑥ [全部接受][部分][拒绝][回滚]│
├────────────────────────────────────────────┴──────────────────────────────┤
│ 📜 提交记录(默认折叠):#3 清洗销售表 ✓ · #2 改公式 ✓ ↩可回滚                  │
└─────────────────────────────────────────────────────────────────────────────┘
```

- **左上工具条 = 红笔批注语言**(⭕圈/➡️箭头/✏️删/〰️重写/❓问),不只是"圈选"。
- **圈选即显示上下文**(人话:"你圈的:销售数据 12行5列"),给"它懂我指哪"的信心。
- **diff 卡片像 PR**:每项含 before/after + 原因(接教学模式③),逐项 ✓/✗。
- **底部提交记录 = Git-like 历史**,可逐次回滚。

### 首发杀手用例(第一刀)🔒

**Excel 财务/运营表清洗与校验**。示例:
> "圈选这张月度销售表——统一日期格式、修复被存成文本的数字、标红异常值、补齐毛利率公式、生成问题清单;先别提交,让我逐项确认。"

为什么是它:需要圈选 · 需要结构化 ChangeSet · 天然需要 diff 与逐项接受 · 容易验证对错 · 体现外科写回保真 · 新手与专业用户都受用 · 比生成 PPT/复杂 Word 排版更易做可靠。
**Word 仅作第二演示**:"圈这几段改成正式公文语气,保留脚注/图片/页眉页脚,给我逐段 diff"——证明抽象层非 Excel-only,但不作主战场。

## 7. 五个创新点 🔒

| # | 创新 | 价值 |
|---|---|---|
| ① | **红笔批注语言** | 圈/箭头/删除线/波浪线/问号 = 人人会的校对符号;降低门槛 |
| ② | **示范即技能** | 做一遍/改一次 Agent 输出 → 自动蒸馏成可复用技能,社区/单位沉淀做法 |
| ③ | **教学模式** | 改动附"为什么+下次怎么做",可切"教我做";把"替你做"升级为"让你会" |
| ④ | **技能即 MCP 工具** | 每个技能可被外部 Agent 调用 → 办公技能基础设施 |
| ⑤ | **文档=可重跑意图程序** | "一次定义、每期自动重做"(周报/对账/上报)= "想法成真"刚需 |

## 8. Agent 编辑管线(可靠性)🟡

意图 + 选区快照 → 模型(BYOK/本地)tool-calling 产出**受 JSON Schema 约束的 ChangeSet**(结构化编辑指令,非裸 OOXML、非瞎改 DOM)→ 服务端 headless **影子应用** + **高保真引擎/公式 parity 校验** → 结构化 diff → 三级粒度(整批/块/单格)接受/拒绝 → 经底座命令系统提交(带 undo/审计)。

## 9. 许可证 🔒

**Apache-2.0**(🔒 已定)。permissive、最大化采用与可嵌入性,且含**显式专利授权**——别人基于它二次商用更安心。
- 代价:不能捆绑 AGPL 组件(OnlyOffice/PPTist/Collabora 编辑器)→ Word 编辑面改用 **TipTap(MIT)/ Univer Docs(Apache)**,失去 OnlyOffice 现成 AI 插件蓝本(Word 线工作量增加)。
- **不受影响**:**LibreOffice headless 以独立子进程调用**(MPL/LGPL,不链接即不传染)做高保真;**外科 OOXML 补丁**许可中立。→ **高保真仍免费可得**。
> 最终以实际依赖的各组件许可为准,商用前请核对。

## 10. 推理成本与新手接入 🟡

- **BYOK / 本地模型** → 项目零推理成本。
- ⭐ **新手最后一公里**:完全不懂技术者不会自配 key。接入路径三选一(可并存):**本地模型一键包** / **学校·单位·技术者部署好供其使用** / **"配一次模型"向导**。"让所有人用"卡在此处,必须专门设计。
- 中国线须接**国产可合规模型**(备案/审核/ICP)。

## 11. ⭐ 动工前必跑的两个杀手实验(1 周)

1. **保真度**:20 个真实 .xlsx + 20 个真实 .docx,过 LibreOffice 子进程 + 外科补丁,测导入导出丢失率(详见 `kill-experiments.md`)。
2. **意图覆盖 + 交互**:收 100 条真实自然语言指令,测"ChangeSet DSL + 技能"覆盖率;观察用户**主动圈选 vs 打字**。

> 任一证伪即重定向。先花一周,省半年。

## 13. 可持续性 🟡

维护者 + 社区贡献 + 捐赠/赞助(GitHub Sponsors / OpenCollective)/ 开源基金。零运营推理成本(BYOK/本地模型)。

## 14. 当前主要风险

- **技术**:OOXML 保真往返、圈选锚点漂移、Univer 公式≠Excel(LibreOffice 重算校验)、ChangeSet DSL 能力天花板、大文件性能、人↔Agent 并发(需自研文档合并/rebase)。
- **可持续性**:无收入,靠维护者时间 + 社区。
- **新手接入**:模型最后一公里(见 §10)。
