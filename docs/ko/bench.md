### 수정 사항 전체 목록(6라운드 누적)
1. 명확화 하드 기준(prompt·라우팅 ③): 목표가 빠진 요청은 반드시 ask_user
2. STEP_LIMIT 8→12(loop): 전문가 플로우의 스텝 수 예산
3. 이어서 배치 선언(prompt·라우팅 ⑥): 이어지는 배치 턴에서는 배치를 선언하고 이미 기록된 항목을 반복하지 않아야 함
4. 차트 유형 기본값(prompt·excel ④): 범주 비교는 기본 bar, "비중"이라고 말하지 않았으면 pie를 쓰지 않음
5. **readRange 타입 가시성(도구 계층)**: 텍스트형 숫자를 명시적으로 표시——핵심 정확성 정보는 스킬 매칭에 의존하지 말고 도구 출력 자체에 드러나야 함
6. **NUDGE_TOOLIFY(채널 계층)**: 맨 텍스트 마무리 시 한 번 도구화를 촉구——라우팅 계약 "매 턴 한 번의 도구 호출"을 코드가 최종 보장
7. bench 앵커 어서션(테스트 자체): opsMust가 anchors를 커버

### 방법론 정리
- 수정은 세 계층으로 나누어 적용: **프롬프트**(행동 선호) → **도구 출력**(정보 가시성) → **채널 코드**(계약 최종 보장); 뒤쪽 계층일수록 확실함
- judge는 미려한 산문에 속아 점수를 주지만 객관적 불변량은 속지 않는다——두 층 모두 필요
- 단일 라운드 점수에는 분산이 있다(x-anomaly가 5/2/5/4 사이에서 요동), 두 라운드 연속 재현되어야 진짜 약점으로 인정
