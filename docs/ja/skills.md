# スキル

`packages/skills` はナレッジハブです。エージェントが*上手にこなせること*を、コアプロンプトの外に
保持することで、すべてのリクエストを肥大化させずに成長させられるようにしています。

## 2 種類の組み込みスキル

| 種類 | 例 | 内容 | 注入形態 |
|---|---|---|---|
| **ケイパビリティカード** | `xlsx`、`docx`、`pptx`、`pdf`、`drawio` | 「このフォーマットが何をサポートするか」の 1 行説明 | L0: システムプロンプト内の名前 + 説明 |
| **プレイブック**（打法手册＝実践手順書） | `docx-gongwen`、`xlsx-financial`、`chart-selection` | チェックリスト + changeset イディオム + アンチパターン | L0 カードに【有打法手册】（プレイブックあり）のタグ付き。全文は `load_skill` によりオンデマンドで取得 |

組み込みプレイブックは **実体のある SKILL.md ファイル**として `packages/skills/skills/<name>/SKILL.md`
に配置されています（Anthropic Agent Skills のディレクトリ規約 — スキルごとに 1 フォルダ、L0 用の
YAML フロントマター、L1 用の markdown 本文）。`playbooks.ts` は単なるローダーであり、markdown を
編集すればよく、コードの変更は不要です。

- **`docx-gongwen`** — GB/T 9704 に準拠した公文（中国の公用文書）レイアウト: タイトル/本文のフォント
  サイズ体系（タイトルは二号小标宋＝二号・小標宋体、本文は三号仿宋＝三号・仿宋体）、
  一、/(一)/1./(1) の見出し番号階層、全角句読点、およびそれらを実現するための changeset イディオム
  （`all=true` の一括適用の前に、実際の見出しには `block` を使う）。
- **`xlsx-financial`** — 突合チェック（合計は数式で算出し、ハードコードしない）、金額/パーセントの
  数値書式、異常値検出、そして絶対のルール: 数値に関する結論を書く前に必ず `read_range`/`aggregate`
  で検証する。原始数据（元データ）を上書きしてはならない。
- **`chart-selection`** — 「問い → チャートタイプ」の決定木に加え、プロフェッショナルとしての最低限の
  ルール（ゼロベースラインの棒グラフ、色は 6 色以下、意味のあるソート順、結論を語るタイトル）。

## 段階的開示（Progressive disclosure）

1. `SkillLibrary.match(intent, format)` がカードをスコアリングします: フォーマット一致で +3、
   キーワード一致ごとに +1、さらに**キーワードが実際に一致した場合に限り**プレイブックに +0.5 の
   タイブレーク（汎用的なインテントでは依然として素のケイパビリティカードが上位になります）。
2. `render()` が上位カード（L0）をシステムプロンプトに注入します。プレイブックカードには明示的な
   指示が付きます: *関連する場合は、行動する前に `load_skill` で読み込むこと*。
3. `load_skill` ツール（`Agent.withSkillTools` が `RespondOptions.extraTools` 経由で接続）は
   プレイブックの markdown 全文をツール結果として返します — 知識は必要になったときにのみ届きます。

## 外部スキル

業界固有・チーム固有のものは組み込みには含めません。ホストは標準的な `SKILL.md`
（Anthropic Agent Skills 互換 — YAML フロントマター + markdown 本文）を実行時にインストールできます:

```ts
library.install(skillMdText, 'file:./skills/my-company-report.md');
```

パースされたカードは、直ちにマッチング/レンダリング/`load_skill` に参加します。L2（実行可能スクリプト）は
意図的に有効化していません — テキストのプレイブックだけで、サンドボックスリスクゼロのまま価値の大半を
提供できるからです。

## 良いプレイブックの書き方

- **モデルが行動前に実行するチェックリスト**から始める — 処方箋よりも診断が有効です。
- **changeset イディオム**を含める: どの ops を、どの順序で使うか（例:「まず `block` で実際の
  見出しを設定し、その後 `all=true` で本文のベースラインを揃える — さもないと一括適用が
  タイトルを潰してしまう」）。
- **アンチパターン**を含める（「手動の太字+サイズ変更で見出しを偽装しない」）。
- 約 50 行以内に収める。ライブなコンテキストに読み込まれるため、密度が勝敗を決めます。
