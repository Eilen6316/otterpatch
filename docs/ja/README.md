## ワンパラグラフの売り込み

エージェントがあなたのファイルを直接編集すべきではありません。OtterPatch では、エージェントは構造化された
`ChangeSet` を**提案（propose）**するだけです。システムはそれをシャドウコピーに対して検証し（モデル自身に
自分のミスを修復させ）、**レビュー可能な diff** — ワークスペース内ではインラインの変更履歴、レールでは
git スタイルの diff — を表示し、項目ごとの人間による承認を経て初めて**外科的（surgical）に**書き戻します。
つまりファイル内で触れられた部分だけが変更され、それ以外はバイト単位で同一のまま保たれます。

## コア不変条件（絶対に破らないこと）

1. **単一の変更出口** — すべてのドキュメント変更は `propose_changeset` を通ります。他のいかなるツールもドキュメントを変更しません。
2. **コミット前のレビュー** — 項目ごとの人間による accept/reject なしには、何もファイルに反映されません。
3. **外科的な書き戻し** — 触れていない部分はバイト単位で同一のまま保たれます。忠実度（fidelity）は計測され、報告されます。
4. **直列書き込み** — 提案は現在のドキュメント状態に対してアンカーされます。バッチは直列に継続され（並列ライターは決して存在しない）、処理途中でアンカーが陳腐化することはありません。
