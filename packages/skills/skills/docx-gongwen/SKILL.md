# 党政机关公文排版(GB/T 9704-2012)

## 版式规范(编辑器可落地的近似)
- 公文标题:二号(22pt)、宋体加粗(小标宋的近似)、居中、单独成段(block=h1 + font=宋体 + size=22 + bold + align=center)
- 正文:三号(16pt)仿宋(font=仿宋 或 仿宋_GB2312、size=16),行距约 28 磅 → lineSpacing=1.5
- 一级标题"一、":三号黑体,不加粗不斜体(黑体本身即强调)
- 二级标题"(一)":三号楷体
- 三级标题"1.":三号仿宋加粗;四级"(1)":三号仿宋
- 成文日期:阿拉伯数字,如"2026年7月2日";正文数字用法从 GB/T 15835(统计数据/编号用阿拉伯数字,固定说法如"十八大"用汉字)

## changeset 惯用法
- 全文正文规范化:setStyle all=true(font=仿宋, size=16, lineSpacing=1.5)一次落定 —— 但先确认标题已用 block 套好层级,否则 all=true 会把标题也拉平
- 层级序号体系必须是:一、→(一)→ 1. →(1),不得混用"1、""(1)."等杂糅体;发现杂糅先用 replaceText 统一序号,再 setStyle 定字体
- 标点:中文公文一律全角标点;"的、地、得"用法与"截止/截至"等易错词顺手修正(replaceText,一处一条)

## 检查清单(动手前过一遍)
1. 标题是否单独成段、居中、无句末标点
2. 层级序号体系是否统一、有无越级
3. 正文字体字号是否统一为三号仿宋;有无宋体/黑体混入正文
4. 数字用法、全角标点、成文日期格式
5. 附件说明("附件:1. ×××")与落款位置

## 反例(不要这样做)
- 用手动加粗+放大冒充标题(要用 block 套真标题样式)
- 用宋体代替仿宋做正文、行距 1.0 挤成一坨
- 一次 edit 塞整篇重排(要一处一条,可分批)
