# 财务/经营数据表规范(Excel)

## 检查清单(诊断优先于动手)
1. 勾稽关系:合计=分项之和(先 aggregate 验证再下结论)、期末=期初+本期增减;三表场景:资产=负债+所有者权益、净利润流向未分配利润、现金流期末数=货币资金
2. 合计行/列必须是公式(=SUM(...)),不得硬编码数字 —— 发现硬编码合计,改成公式并在 plan 说明
3. 数字格式:金额千分位(#,##0.00)、百分比两位小数(0.00%)、负数用红色括号((#,##0.00));同列格式必须一致
4. 异常侦测:环比/同比突变(>30% 值得标注)、毛利率为负、费用率异常、重复行、空值断档;文本冒充数字(如 "71")会让 SUM 漏算,必须转数或提醒

## changeset 惯用法
- 改数先查数:任何"改成正确的值"之前,用 read_range 取真实区间、aggregate 验证口径,不许凭样本臆测
- 公式优先:能用 =SUM/=AVERAGE 表达的绝不写死算后值(写死的值在源数据变化后会变成错数)
- 标异常:bgColor 浅红(#ffd6d6)+ 该行加说明,不动原值 —— 分析层与数据层分离
- 大表分批:一次 ops 过多会被截断,plan 里声明"先前 N 行"

## 红线
- 不改原始数据区的历史数值(那是凭证);只修公式错误、加分析层、规范格式
- 改公式前 read_range 看清它引用的区域,别把依赖链改断
