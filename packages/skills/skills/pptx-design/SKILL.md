# 幻灯片设计打法(PPT)

## 配色纪律
- 一个主色占 60-70% 视觉权重 + 1-2 个辅色 + 1 个锐利强调色;三色均分是业余感的来源
- 明暗三明治:封面/结尾深底、内容页浅底(或全深走高级感);浅底浅字、深底深图标这类低对比组合是硬伤
- 强调色只给"要说的那一个"元素,别满页都是重点

## 字号与字体体系
- 页标题 36-44pt 粗体 / 节标题 20-24pt / 正文 14-16pt / 注释 10-12pt——标题与正文拉不开层级等于没有层级
- 标题字体可以有个性,正文字体必须干净;全篇最多两种字体
- 数据页用大数字(60-72pt)做主视觉,配一行结论

## 版式与母题
- 版式库轮换:双栏对比 / 图标+短句列表 / 2×2 网格 / 半版出血图+文字叠压 / 时间线;全篇复用同一版式是败笔
- 选定一个视觉母题(圆角图框/彩色圆底图标/单侧粗边)全篇贯彻——要么全篇统一,要么全篇极简,只精修一页最难看
- 每页至少一个视觉元素;纯文字满屏页是要被改掉的对象
- 边距 ≥ 半英寸,块间距全篇统一;标题按单行设计时想想它折行后装饰元素会不会错位

## AI 味禁则
- 标题下加一根强调短线 = AI 生成的标志性破绽,禁用(用留白或色块层次替代)
- 默认蓝 + Arial + 居中正文 = 三连破绽
- 占位文本(lorem/xxx/模板示例句)残留 = 未完成,交付前必须逐页 grep

## QA 心态
- 默认首版有错:元素重叠、文字溢出截断、页脚撞正文、间距忽宽忽窄、对齐不齐、文本框过窄疯狂换行——按这张清单逐页过,一轮检查零发现说明没认真看
- 改一处会引发新问题:修完必须复看受影响页,至少完整跑一轮"修→验"再宣告完成
